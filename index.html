MediaFire API Website

Dapatkan informasi dan link download dari MediaFire

API Endpoint

GET /api/mediafire?url={mediafire_url}

Gunakan endpoint ini untuk mendapatkan informasi file dan link download dari MediaFire.

Contoh Penggunaan:

curl -X 'GET' \
'https://your-vercel-app.vercel.app/api/mediafire?url=https%3A%2F%2Fwww.mediafire.com%2Ffile%2Fvj3al1c98u2zdr6%2FTerakomari_-_MD.zip%2Ffile' \
-H 'accept: application/json'

Response Sukses:

{
  "success": true,
  "data": {
    "name": "Terakomari - MD.zip",
    "size": "6.78MB",
    "extension": "zip",
    "uploaded": "2024-02-22 07:29:23",
    "downloadUrl": "https://download2264.mediafire.com/.../Terakomari+-+MD.zip"
  }
}

Penanganan Masalah Umum

1. Download URL Kosong

Kadang-kadang MediaFire mengembalikan downloadUrl yang kosong. API ini akan:

  • Mendeteksi ketika downloadUrl kosong
  • Secara otomatis mencoba mendapatkan kunci unduh baru
  • Menggunakan endpoint perbaikan MediaFire

2. Perlu Generate Key Baru

MediaFire kadang memerlukan generate key baru untuk download:

  • API akan otomatis mengikuti proses generate key
  • Menangani penundaan 5 detik yang diperlukan
  • Mengambil download URL yang baru digenerate

3. File Gambar

Untuk file gambar, MediaFire menggunakan URL yang berbeda:

  • API mendeteksi ketika file adalah gambar
  • Menggunakan endpoint convkey untuk mendapatkan gambar asli
  • Mengembalikan URL gambar yang dapat diakses langsung

Cara Kerja API

API ini menggunakan beberapa teknik untuk mengatasi pembatasan MediaFire:

  1. CloudScraper - Untuk melewati perlindungan CloudFlare
  2. Puppeteer - Untuk menangani halaman yang memerlukan interaksi JavaScript
  3. JSDOM - Untuk parsing HTML dan ekstraksi data
  4. Retry Mechanism - Untuk mencoba kembali ketika gagal

API akan secara otomatis mendeteksi skenario yang berbeda dan menerapkan penanganan yang sesuai.
